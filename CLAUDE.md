## Project Overview

Sarkaz Cipher Decoder for the game *Arknights: Endfield*. Decodes English letter strings back into Chinese text using a modular arithmetic cipher.

**Cipher rule:** `Letter = REMAINDER_MAP[ord(Chinese_Character) % 56]`

The `REMAINDER_MAP` is a 56-element list. Because multiple remainders can map to the same letter (e.g., both 0 and 27 → `'g'`), decoding is one-to-many: each input letter maps to a set of candidate Chinese characters.

## Running

```bash
python main.py
```

No dependencies beyond the Python standard library. No build step, no virtual environment required.

## Architecture

The project is currently a single-file prototype (`main.py`) with a planned 3-module expansion:

| Module | Purpose |
|---|---|
| `scraper.py` | Scrape Endfield proper nouns from `wiki.skland.com/endfield` → `endfield_dict.txt` |
| `build_dict.py` | Build reverse-lookup JSON `{"gds": ["陈千语", ...]}` from scraped + GB2312 Level 1 chars |
| `decoder.py` | DP/Viterbi or Trie segmentation to decode continuous letter strings into coherent sentences |

**Priority order for decoding:** Endfield terms > common Chinese words > single characters.

## Data Files

- `remainder_map.csv` — the cipher table (remainder 0–55 → letter). Auto-generated by `ensure_map_csv()` if missing.
- `possible_chars_filtered.csv` — output: per input letter, lists matched remainders and candidate GB2312 Level 1 characters.

## Key Implementation Detail

`get_common_chinese_characters()` in `main.py` extracts the 3755 GB2312 Level 1 characters by iterating zone codes 16–55 and position codes 1–94, constructing raw GB2312 bytes and decoding to Unicode. This avoids any external dictionary dependency.
